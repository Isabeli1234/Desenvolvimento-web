CSS inline:

A primeira é o CSS inline, aplicado diretamente no elemento HTML por meio do atributo style.
Sua principal vantagem é a agilidade, pois permite mudanças rápidas e pontuais sem a necessidade de alterar
outros arquivos. No entanto, ele dificulta a manutenção, polui o HTML e não possibilita o reaproveitamento
de código. O motivo mais importante para utilizá-lo é justamente a agilidade máxima
para modificar um elemento específico.

CSS interno:

A segunda é o CSS interno, no qual as regras ficam dentro de uma tag style no cabeçalho head da
página. Esse formato centraliza os estilos de cada página em um único local, sendo útil para ajustes e testes
rápidos em páginas isoladas. A desvantagem é que não permite o compartilhamento de estilos entre
diferentes páginas e pode se tornar confuso em projetos maiores. Seu motivo mais importante é o controle centralizado
de todos os estilos da página.

CSS externo:

A terceira forma é o CSS externo, que utiliza um arquivo separado normalmente com extensão .css
vinculado ao HTML por meio da tag link. Ele mantém o HTML limpo, facilita a manutenção, possibilita o
reaproveitamento de código e melhora o desempenho com o uso de cache. A principal desvantagem é
exigir mais organização e estrutura inicial, além de gerar uma requisição a mais ao carregar. O motivo mais
importante para usá-lo é a manutenção fácil e consistente
em todo o site, permitindo que várias páginas compartilhem o mesmo estilo.
